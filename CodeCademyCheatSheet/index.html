Semantic HTML Cheatsheet

What is Semantic HTML?

Semantic HTML, also known as semantic markup, refers to the use of HTML tags that convey the meaning—or semantics—of the content contained within them.

By adding semantic HTML tags to your pages, you provide additional information that helps define the roles and relative importance of the different parts of your page.

(As opposed to non-semantic HTML, which uses tags that don’t directly convey meaning.)

Common Semantic HTML Elements

  • <header>:Represents the introductory content or a set of navigational links at the top of a document.

  • <nav>: Represents a section of navigation links.

  • <article>: Represents a self-contained composition in a document.

  • <section>: Represents a standalone section of content within a document.

  • <aside>: Represents content tangentially related to the content around it.

  • <footer>: Represents the footer of a section or a page.

  • <figure> and <figcaption>: Used for embedding media and providing a caption.

  • <main>: Represents the main content of the document.

What are some benefits of using semantic HTML?

  1. Accessibility: Semantic HTML helps make your website more accessible to individuals with disabilities. Screen readers and other assistive technologies rely on the semantic structure of the page to provide a meaningful experience to users. For example, using semantic headings (<h1>, <h2>, etc.) and lists (<ul>, <ol>, <dl>) helps screen readers provide context and navigation.

  2. Search Engine Optimization (SEO): Search engines use semantic HTML to better understand the content and structure of your web page. Using proper headings, paragraph tags, and other semantic elements can improve your page's ranking in search engine results.

  3. Maintainability: Semantic HTML makes your code more readable and maintainable for developers. It's easier for other developers to understand the structure and purpose of your page when you use semantic elements. This can lead to faster development, easier updates, and reduced likelihood of introducing bugs.

  4. Consistency: Semantic HTML encourages consistency in the structure and layout of your web pages. It helps you follow best practices for web development and maintain a cohesive design across your site.

  5. Future-proofing: Semantic HTML is less likely to become deprecated or obsolete as web standards evolve. Using semantic elements ensures that your content remains structured and meaningful as new technologies and standards emerge.

  6. Improved User Experience: A well-structured, semantic document provides a better user experience. It helps users easily navigate and understand the content, which can lead to increased engagement and satisfaction.
